# Project Structure

```
.
├── README.md          # Project goal + table of contents + command usage + samples
├── !coding-standards/ # Reference coding standards
├── Makefile           # install-prerequisites, all, clean, build, flash, test...
```

- `workflows/` contains plain-English recipes that the agent follows.
- `docs/` contains reference documentation and templates.
- `output/` is where deliverables are saved.
